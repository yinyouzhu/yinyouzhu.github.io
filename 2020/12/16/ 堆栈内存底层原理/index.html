堆栈内存底层原理

JS基础知识
JS基础知识
发布日期:   2020-12-16
文章字数:   1.1k
阅读时长:   4 分

提到堆栈内存，大家首先想到的应该是C语言中的堆栈内存。其实JS中也是存在堆栈内存的。下面这篇文章带大家了解下JS中的堆栈内存。

特性

堆内存存储的是引用数据类型 栈内存存储的是基本数据类型

名词解释&执行过程

1. ECStack: JS想要执行代码，一定会开辟一个执行栈，也叫栈内存，专业名词，ECStack

EC(G): 创建全局执行上下文EC(G),某个域下的代码执行都有自己的执行上下文，全局EC(G) 

2. 把创建的EC执行上下文，压缩到栈中执行(进栈),执行完有的上下文就没有用了，这个时候出栈，有的还要用，会把其压缩到栈底(闭包) 

3. GO(golbal object)全局对象，在浏览器端(非node)，会把全局对象赋值给window，window指向GO， window = {xxxx:xxxx} 

4. VO(Varibal object)变量对象：在当前上下文中，用来存放创建的变量和值的地方(每一个执行上下文中都会有一个自己的变量对象，函数私有上下文中叫做AO，但也是变量对象)，函数执行形成的变量对象是AO，全局执行形成的变量对象叫VO 

基本数据类型创建变量过程
例如 var a = 12 (基本数据类型都是直接存储到栈内存中) 
1. 创建值，存储到栈内存中 -> 创建12，并且存储到栈内存中
2. 声明变量   -> 声明变量a 
3. 让变量和值关联起来(赋值) ->  a = 12  

引用数据类型创建过程
如 var a ={name:'小明'} 
1. 开辟一个堆内存，并且有一个16进制的内存地址   
2. 把键值对存储到开辟的堆内存中 
3. 将堆内存的地址存放到栈内存中，供变量调用(关联) 
4. 创建变量

例

var obj = {
    name: '测试',
    fn:(function (x) {
        return x + 10;
    })(obj.name)
};

运行结果为报错，原因如下
首先我们根据引用数据类型创建过程来分析报错原因
1. 创建一个堆内存，并且有一个16进制的内存地址
2. 把键值对存储到堆内存中  name:'测试'
3. 将创建好的堆内存地址存储到栈内存中
因为这个是一个自执行函数，所以是需要把obj.name的值当作实参传进来，但是因为堆内存中的信息还没有存储完成，obj和堆还没有建立关联，此时的obj是undefined，所以不能做成员访问

浏览器的垃圾回收机制

谷歌： 浏览器会在空闲的时候,把所有不被占用的堆内存，进行释放和销毁 

IE：当前堆被占用一次，记数字1，再被占用一次，数字再累加1，如取消占用，则数字减1，直到减到0为止

引用类型和基本数据类型区别

基本数据类型
let a = 15;
let b = a;
b = 10;
cosole.log(a,b)  // a = 20 b = 10 

那么为什么是这个结果呢？
因为 a 和 b 都是基本数据类型，是存储在栈内存中的。对于 b 重新赋值，并不会影响到 a 的值。下面的例子会演示引用数据类型
引用数据类型
let obj = {
    a:10
}
let valueA = obj;
valueA.a = 20;
console.log(value.A,obj.a) // valueA.a = 20  obj.a = 20   

解: 
1. 首先我们定义了一个obj,因为obj是个对象，所以是引用数据类型，是存储在堆内存中的。此时这个堆内存会有一个16进制的内存地址，并且赋值给变量obj，也就是说，此时obj存储的是这个堆内存的内存地址，我们暂且给他定义为AAAFFF000, 
2. 此时我们又创建了一个valueA = obj, 此时的操作是把之前创建的堆内存地址指向到了valueA的这个变量上，相当于valueA指向的内存地址也是AAAFFF000 
3. valueA.a 这一步中，因为valueA指向的是地址为AAAFFF000的堆内存，而且这个堆内存中存储了 a:10 这个键值对，所以此时的操作就是修改了AAAFFF000中的这个属性a，并且重新赋值为20.  
4. 因为obj和avlueA同时指向了AAAFFF000这个堆内存。所以此时valueA.a 和 obj.a 的值都为20
